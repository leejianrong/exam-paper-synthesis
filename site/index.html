§ 01 — The problem

Making a worksheet by hand takes hours. Making one with AI costs you trust.

The part of a maths worksheet that matters most — the maths — is the part most likely to carry a silent mistake.

By hand

Slow, and still error-prone

Write each question, draw every bar model and figure to scale, work out each answer, then lay out a matching answer key. An evening's work for one topical worksheet — and one slip in the key you won't catch until a student does.

By LLM

Fast, and confidently wrong

Generic "worksheet generators" either recycle a fixed bank (questions repeat, can't be nudged) or let a model invent the maths — which produces plausible, wrong answers and diagrams that don't match the numbers.

A single wrong answer key destroys a teacher's trust in the whole tool.

§ 02 — Why you can trust it

The answer isn't generated. It's computed — then checked.

No model is asked to do arithmetic. Every question runs through the same deterministic pipeline, seeded and reproducible, and nothing is shown until it passes.

01 / sample

Sample

Draw parameters from the blueprint's declared schema. Seeded, so the same seed always yields the same question.

02 / solve

Solve

A hand-written Python solver computes the final answer and every intermediate step. This is where the maths lives.

03 / validate

Validate

Assert the constraints hold and that every label in the diagram equals the corresponding solved value.

04 / assemble

Assemble

Build one canonical object: stem, parts, typed answer with units, marks, and worked solution steps.

05 / check

Schema-check

Validate against a versioned JSON Schema. Any error is rejected with a path-pointed message — bad data never gets through.

Correct by construction, regression-tested by hand. Each blueprint ships with golden fixtures — params → expected answer & marks — hand-verified by a person, never checked by a model. Corrupt the solver and a fixture fails loudly.

§ 03 — What you get

A worksheet you'd be happy to put in front of a class.

Built for the way tuition teachers actually prepare P5–P6 Standard practice.

A full set in minutes

20–50 fresh questions per topic in one batch — every generation a new instance with different numbers, so worksheets never repeat.

Recognisably PSLE

P5–P6 Standard, syllabus-aligned, including the multi-step, non-routine word problems the real exam demands — with multi-part questions that build.

Diagrams that match

Bar models and geometry figures drawn from the same numbers as the question — every dimension and label is checked against a solved value.

You have the final say

A human review gate sits between generation and export. Approve, regenerate, or discard each question — nothing goes out that you haven't cleared.

Two clean PDFs

Export a printable worksheet — numbered questions, [n] marks, answer spaces — and a separate answer-key PDF with the worked steps.

Nudge the difficulty

Make a question harder or easier, swap to decimals, or toggle the bar-model aid — each edit a new version, linked back to the original.

§ 04 — One object, two views

The worksheet and the answer key are the same truth, printed twice.

Both PDFs are rendered from one canonical question object — so the key can never drift from the question.
